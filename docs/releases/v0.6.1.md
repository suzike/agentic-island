# Agentic-Island v0.6.1

`v0.6.1` 聚焦大模型接入可靠性和问答会话完整度，修复 DeepSeek、Kimi、Claude 配置兼容、异步响应串线、分支上下文泄漏与长会话持久化问题，并保留每条回答气泡内的独立继续追问体验。

## 主要更新

- DeepSeek V4 按官方协议区分快速与深度模式，正确控制 thinking、reasoning effort、token 预算和 temperature。
- Kimi Code 固定使用官方 Base URL 与 `k3`、`kimi-for-coding`、`kimi-for-coding-highspeed` 模型目录；Kimi 开放平台继续支持在线模型同步。
- Claude 切换为原生 Anthropic Messages API，更新当前模型 ID，支持旧模型配置自动迁移、多模态消息和深度思考参数。
- Base URL 自动清理尾部路径与空白，认证和网络错误统一脱敏，避免密钥出现在界面、日志或异常信息中。
- 模型同步、连接测试与回答请求绑定发起时的供应商、模型和密钥，切换配置后旧响应不会覆盖当前状态。
- 问答分支、Fork、气泡内继续追问、回答分析和多候选结果统一绑定稳定分支 ID，避免异步请求写入错误会话。
- 知识库请求使用发起分支的持续指令快照，防止切换分支后规则串线。
- 长会话压缩持久化会优先保留钉选的重要消息，再保留最近上下文；临时输入态和流式占位不再落盘。
- 问答操作重新围绕会话增强组织，保留分支谱系、上下文选择、多模型对比、知识库核验与气泡内追问。

## 兼容与迁移

- 旧 Claude 模型 ID 会迁移到 `claude-sonnet-4-6`、`claude-opus-4-8` 和 `claude-haiku-4-5`。
- Kimi Code 中保存的无效模型会恢复为官方默认模型，不影响其他供应商的自定义模型。
- 已保存的问答分支、历史消息、附件、长期记忆和持续指令继续兼容。
- DeepSeek、Kimi、Claude 与自定义 OpenAI 兼容服务仍各自保存配置，密钥仅存储在本机加密状态中。
