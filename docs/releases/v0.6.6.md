# Agentic-Island v0.6.6

发布日期：2026-07-23

`v0.6.6` 将问答从“选择模型后直接回答”扩展为“按问题选择回答方法，并可在回答气泡下继续做独立深度分析”。默认回答行为、气泡内继续追问、分支、上下文控制和知识库能力保持兼容。

## 本轮回答方法

- 15 种回答方法：金字塔原理、MECE、费曼解释、苏格拉底式回答、第一性原理、系统思维、科学方法、加权决策矩阵、情景规划、六顶思考帽、JTBD、设计思维、TRIZ、OODA 和 RFC/ADR。
- 方法按表达组织、推理求解、决策判断、创新设计和行动落地分类，并根据当前问题在本地推荐 3 种。
- 未选择方法时保持原有默认回答；所选方法仅影响下一次发送，完成后自动恢复默认。
- 云模型、本地 Claude/Codex、知识库 RAG 和快捷问题统一携带方法指令，问题气泡记录实际采用的方法。

## 回答气泡分析中心

- 23 种分析方法按事实证据、逻辑因果、风险压力、决策取舍和执行验证分类。
- 包含主张-证据-置信度、红队挑战、钢人化、预演失败、FMEA、二阶效应、敏感性分析、因果关系图、5 Whys、约束理论、机会成本、利益相关者地图、反事实检验、情景压力测试、加权决策矩阵和验证与验收计划等方法。
- 分析结果附着在原回答气泡，不新增主消息、不进入后续模型上下文；重复执行同一方法会更新原结果。
- 气泡内继续追问仍保留在每个回答气泡下，不改为主会话消息。

## 交互与验证

- “更多”菜单按上下文、继续使用、沉淀与复用重新组织，并支持引用整条回答。
- 方法卡片在窄面板自动切换为单列。
- `npm test`：36 个离线测试全部通过。
- `npm run typecheck`
- `npm run build`
- `npm run audit:ask`
- `npm run package`
- `npm run verify:package`

## 已知限制

- 当前只支持 Windows 10/11 x64，安装包未使用商业代码签名，Windows SmartScreen 可能显示未知发布者。
- 回答方法用于组织模型输出，不保证模型事实正确；高风险结论仍应使用证据核验、红队或验证计划进行复查。
