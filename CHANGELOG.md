## [0.6.3] - 2026-07-22

### Changed

- PowerShell 工具栏的文件夹入口改为 Windows 原生目录选择器；确认后对路径进行 PowerShell literal 转义，并立即切换当前 ConPTY 会话的工作目录。
- 目录选择器接入统一 External Yield 原生对话框包装，打开期间降低灵动岛最高层级，关闭后恢复。

### Fixed

- 修复 PTY 每次字符回显都会从旧提示符重复识别相同目录、创建新标签数组并触发终端模块重渲染的问题。
- 修复 `ResizeObserver` 连续通知、xterm resize 事件和显式 IPC resize 叠加造成的输入行抖动与 ConPTY 无效重排。
- 主进程现在记录每个 PTY 的最后尺寸并忽略重复 resize；会话退出或关闭时同步清理尺寸状态。

### Verification

- 新增终端目录状态引用稳定性和原生目录选择器 External Yield 接入回归。
- 两套 TypeScript 检查、33 个离线测试、Electron 三端生产构建、真实 Electron 连续输入几何检查、NSIS 打包及隔离安装/启动/卸载验证通过。

## [0.6.2] - 2026-07-21

### Changed

- OpenAI 官方目录升级到 GPT-5.6 Sol / Terra / Luna；快速与深度模式分别映射低/最大 reasoning effort，并改用 `max_completion_tokens`，不再发送旧 `max_tokens` 或不兼容的 `temperature`。
- Claude 官方目录升级到 Sonnet 5、Opus 4.8、Fable 5 与 Haiku 4.5；按型号分别映射 adaptive thinking、effort、显示摘要和 Haiku 手动思考预算。
- 知识库 Embedding 改为独立保存 Base URL、模型和 API Key；旧版只保存模型的配置会一次性迁移当前连接，之后切换问答供应商不再影响 RAG。
- 设置页和问答模型菜单显示连接来源与配置编号；每条新云端回答记录实际请求模型，后续切换模型仍可追溯回答归属。

### Fixed

- 修复同一供应商、模型和端点下的不同账号配置在保存时互相覆盖，以及设置页未比较密钥却错误标记“使用中”的问题。
- 修复仅存在于已保存配置中的模型不会出现在问答切换菜单的问题；保存与载入现在会原子同步供应商、模型、端点、密钥和型号目录。
- 修复多模型讨论把同模型、同端点但不同账号误判为重复配置，以及当前配置不可用时共识/辩论主持请求失败的问题。
- 修复切换到 Anthropic 或 Kimi Code 聊天模型后，知识库仍复用聊天端点而导致向量检索和会话写入失效的问题。

### Verification

- 新增 GPT-5.6、Claude 5、不同账号配置、独立 Embedding 迁移与请求体兼容回归；气泡内继续追问、上下文隔离和分支回写原有测试保持通过。
- 两套 TypeScript 检查、33 个离线测试、Electron 三端生产构建和生产依赖审计通过。

## [0.6.1] - 2026-07-21

### Changed

- 重构问答交互层级：高频操作直接展示，分支、记忆与多模型讨论归入会话管理；操作入口改为带结果说明的文字动作和执行反馈。
- 保留每条回答气泡内的“继续追问”支线；回答分析、多模型候选和汇总结论固定附着到原回答，不再形成难以辨认的主会话消息。
- 新会话和 Fork 增加执行前说明，消息上下文状态改为常驻可见标记。
- Claude 供应商改为原生 Anthropic Messages 协议，支持 `x-api-key` 鉴权、图片输入、模型目录发现和 adaptive thinking；默认目录迁移到 Sonnet 4.6、Opus 4.8 与 Haiku 4.5。
- DeepSeek V4 快速/深度模式现在真实控制 thinking 与 reasoning effort；Kimi K3 映射 low/max effort，Kimi Code 固定使用三个官方 Model ID。

### Fixed

- 修复追问、回答分析或多模型讨论期间切换分支时，异步结果可能按消息序号串写到另一会话的问题；结果现在按稳定分支 ID 回写发起分支。
- 修复主回答、知识库回答、记忆压缩和分支合并在切换会话后仍可能写入当前分支的问题；同一分支生成期间现在会阻止重复发送，其他分支仍可独立使用。
- 修复本机 Claude/Codex 依赖 CLI 全局最近会话、无法可靠遵守岛内分支上下文的问题；两种引擎现在显式携带岛内历史、记忆和会话规则。
- 修复置顶上下文在主进程请求层被二次截断、历史附件无预算增长、归档会话残留永久执行中状态的问题。
- 修复删除父分支后子分支保留失效谱系，以及本机 Agent 停止后部分输出可能被误标为正常完成的问题。
- 修复知识库检索等待期间切换会话会误用另一分支规则，以及历史截尾会丢失较早“重要内容”的问题；持久化现在在 60 条限额内优先保留重要消息。
- 修复同一供应商内修改模型、地址或密钥时，旧的连通测试/模型同步响应仍覆盖当前配置的问题；同端点不同账号的已保存模型也可在问答区切换。
- 修复误填完整 `/chat/completions` 或 `/messages` 地址、复制配置带首尾空白后请求失败的问题；HTTP 错误体和网络异常统一按当前密钥脱敏。
- 将 Kimi For Coding 会员服务与 Kimi 开放平台拆分为独立供应商，分别使用 `api.kimi.com/coding/v1` 和 `api.moonshot.cn/v1`，修复会员密钥被发送到开放平台后返回 401 的问题。
- 新增 `kimi-for-coding`、`kimi-for-coding-highspeed`、`k3` 官方模型目录，并迁移旧版手动 Kimi Code 配置；旧 Kimi 密钥会一次性预填到新入口，原开放平台配置仍保留。
- Kimi 请求改用 `max_completion_tokens` 并省略受思考模式约束的显式 `temperature`，避免认证通过后继续触发模型参数错误。

### Verification

- 问答回归新增气泡内追问、分析结果附着、同类分析替换、生成锁定、临时状态清理、本机 Agent 上下文、附件预算和主上下文隔离检查；隔离 Electron 实例已重建并人工检查问答真实截图。
- 新增 Kimi 双端点、配置迁移和请求体兼容回归；两套 TypeScript 检查、33 个离线测试和 Electron 三端生产构建通过。
- 新增 Anthropic 原生请求、DeepSeek thinking、Kimi K3 effort、错误脱敏、Base URL 归一化、Claude 模型迁移和重要上下文持久化回归。

## [0.6.0] - 2026-07-21

### Added

- 问答新增会话分支树：可从任意提问或回答 Fork，保留父子谱系、独立命名、快速切换，并可将其他分支压缩后合并进当前长期记忆。
- 新增会话级长期记忆和持续指令；消息可钉选为永久上下文或排除出模型请求，文本附件会在后续轮次持续生效，并提供真实上下文/token 预算检查。

- 新增回答自检、假设与风险、替代路径、问题拆解、苏格拉底式澄清、知识库核验和下一问建议。

### Changed

- 问答空态推荐与常驻迷你条改为同源动态内容池，复用启用主题、AI/内置/GitHub/工作简报内容和轮播节奏。
- DeepSeek、Kimi 等供应商改为独立持有模型、Base URL 与 API Key；支持旧配置迁移、在线 `/models` 目录同步和账号实际可用模型切换。

### Fixed

- 修复打包后审计/安装验证实例忽略 `AIISLAND_BRIDGE_FILE`、仍可能覆盖真实 bridge discovery 的隔离缺口。

### Verification

- 新增并扩展问答与供应商回归测试，覆盖 Fork 截断、长期记忆、上下文钉选/排除、附件连续上下文、知识库 Markdown 序列化、旧配置迁移、供应商密钥隔离和目录升级。
- 两套 TypeScript 检查、32 个离线测试、Electron 三端生产构建和生产依赖审计通过。
- 8 张隔离 Electron 真实截图已重建；NSIS 安装包、unpacked 启动、静默安装启动、隔离 bridge 与静默卸载验证通过。
