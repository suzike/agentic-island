1 · 产品概述

Agentic-Island 是一款面向 Windows 的桌面常驻工具，把「灵动岛」形态的信息岛移植到 Windows。它常驻于主屏顶部中央，用来集中监控和管理多个 AI 编码 Agent（Claude Code、Codex 的 CLI 与桌面端）的运行状态，并在 Agent 需要人类介入时——请求权限、提交计划、等待回答——直接在这个岛上完成审批、接力对话与知识问答，而无需切回终端或各自的桌面端。

核心价值主张：把「人类在多个 Agent 之间的上下文切换成本」压缩到接近于零。工程师同时驱动多个 Agent 并行工作时，痛点不是启动 Agent，而是不断被打断、需要在多个终端窗口间来回跳转确认。Agentic-Island 以一个统一的、平时静默、需要时才出现的顶部岛来承载所有这些交互。

它区别于系统托盘图标或通知气泡的地方在于三点：（1）常驻但克制——无事静默、有事才现身；（2）可操作而不仅是通知——审批、对话、问答都能就地完成；（3）单一聚合入口——所有 Agent 会话集中在一个面板里一览。

2 · 背景与竞品移植挑战

灵感来源是 macOS 上的同类产品：它利用 Mac 的「刘海（notch）」区域承载一个可展开的信息岛，集中管理 AI Agent 的权限审批、终端跳转、计划审阅与声音提醒。该形态在 Mac 上天然成立，因为刘海是一块系统预留、视觉上「本就存在」的黑色区域。

移植到 Windows 的核心挑战：Windows 没有刘海。因此不能照搬「贴合刘海」的形态，必须重新设计岛的落点、唤起方式与形态语言。本设计给出的解法是：

  • 虚拟刘海——在主屏顶部中央绘制一个悬浮的圆角胶囊/面板，模拟灵动岛的位置感，但视觉完全原创（毛玻璃material + 终端氛围的荧光绿强调色）。
  • 顶部感应区唤起——把鼠标移动到屏幕顶部中央的一条隐形感应带，即可唤出胶囊；这复刻了「抬头看一眼刘海」的直觉。
  • 三态生命周期——静默（后台无痕）→ 唤出（胶囊）→ 展开（面板），配合「贴住」让常驻可控。

知识产权说明：本产品仅复用「AI Agent 状态监控 + 就地审批」这一产品概念与交互范式，全部图形语言、图标、动效、配色与品牌均为原创，不照搬任何竞品的具体视觉设计。

3 · 目标用户与使用场景

目标用户：初期为作者本人及团队内部工程师；使用效果良好后可对外开源。用户画像是「同时驱动 2–5 个 AI 编码 Agent 并行工作」的开发者。

典型场景：

  • 并行监控——三个 Agent 分别在改后端、写文档、加移动端功能，用户扫一眼胶囊即知「几个在跑、几个待处理」。
  • 就地审批——Codex 请求执行 rm -rf dist/，岛上直接弹出并标红，用户二次确认或拒绝。
  • 不切窗接力——用户认为 Agent 的方案不妥，直接在岛上打字接力对话，让它换方案，无需回到终端。
  • 随手问答——工作中卡壳，直接在岛的「问答」里提问，得到结构化排版的答案。

4 · 目标与非目标

✓ 目标（In Scope）
  • 多 Agent 状态一览（运行中 / 待权限 / 完成）
  • 权限审批 Allow / Deny + 命令风险分级
  • 就地对话接力（Agent 与 Plan）
  • 知识问答助手（接入自定义大模型）
  • 富输入：文字 / 图片 / 文件
  • 终端跳转、快捷键、完成小结
  • 专注模式、贴住、多显示器、开机自启
✕ 非目标（Out of Scope）
  • 替代终端 / IDE 本身
  • 管理 Agent 的模型配置（由各 CLI 自行配置）
  • 语音输入（本期已移除，仅保留文字/图片/文件）
  • 云端同步会话历史
  • 移动端 / Web 端

5 · 形态与交互模型

Agentic-Island 的生命周期是一个三态模型，三态之间平滑过渡：

① 静默态
默认状态，后台运行无任何可见 UI（可选：无待办时完全隐藏胶囊）。顶部中央保留一条隐形感应带作为唤起热区。
② 唤出态
鼠标移入感应带，或有新事件时，胶囊从顶部滑出，显示「N 个 Agent · M 运行中 / 待处理」的摘要。
③ 展开态
点击胶囊，向下展开完整面板（Agents / Plan / 问答 / Settings 四个分区）。

唤起与收起规则（重点交互）：

  • 悬停唤出：鼠标进入顶部感应带 → 胶囊滑出。
  • 点击展开、常驻不闪退：一旦点击展开面板，面板将保持打开，不因鼠标移开而收起——这是为了让用户能安心在里面打字、审批。只有点击面板外的空白处，或再次点击胶囊，或按 Esc，才收起。
  • 事件自动弹出：当有 Agent 请求权限时，面板自动弹出并高亮（除非处于专注模式）。
  • 贴住（📌）：点亮面板头部的图钉后，胶囊常驻显示，不再随鼠标移开而收起，适合盯着一批任务时使用。

6 · 视觉设计系统

整体基调：深色玻璃拟态 + 终端氛围的荧光绿强调色，呼应「编码 Agent」的属性，同时保持精致克制。

要素 规格
主强调色荧光绿 oklch(0.78 0.16 150)，用于运行中状态、主按钮、选中态、开关。
待处理色琥珀 oklch(0.80 0.13 75)，用于「待权限」提醒（同色相衍生）。
危险色红 oklch(0.65 0.20 25)，用于危险命令、久等升级。
面板材质深色半透明 oklch(0.20 0.02 155 / .72) + backdrop-filter: blur(26px) saturate(160%)。
圆角面板 22px、卡片 16px、胶囊全圆角（999px）、按钮胶囊形。
字体正文 Segoe UI / PingFang SC；代码、命令、model id 用等宽 Cascadia Code / Consolas。
面板宽度468px（经用户反馈由 392px 加宽）。
工具图标CLI 用等宽符号 ›_；桌面端用小窗口图标（带标题栏），二者一眼可辨。
滚动条半透明主色细条（7px），并从圆角内收，避免滚到底戳出边框。
动效状态点 2s 呼吸；待处理时面板琥珀高亮呼吸；久等升级为红色、加快至 0.9s；面板展开为即时（避免与逐秒计时器打架，见 §18）。

7 · 信息架构与导航

展开面板由头部 + 四个分区标签构成：

  • 头部：品牌标识、时钟、🌙 专注模式按钮（带倒计时）、📌 贴住按钮。
  • Agents：所有会话卡片列表（有待处理时标签带琥珀点）。
  • Plan：当前计划的 Markdown 渲染 + 审阅 + 接力对话。
  • 问答：知识问答助手，顶部显示当前模型。
  • Settings：通用、问答助手模型、已支持工具三组。

8 · 功能详述：Agents 一览

每个 Agent 一张卡片，包含：状态点（颜色区分运行中/待处理/完成）、工具类型图标（CLI/桌面端）、工具名、项目路径（等宽字体）、状态标签、一句当前动作描述。卡片底部按状态提供操作：

  • 运行中：「↗ 跳转到终端」——点击将系统焦点切到对应的终端窗口（支持分屏、tmux 等，见 §18）。
  • 待处理：权限审批区（见 §9）。
  • 完成：变更小结 + 跳转 + 查看变更（见下）。
  • 所有状态：「💬 继续对话」入口，展开就地接力（见 §10）。

完成小结：Agent 完成任务时，卡片展示一行变更速览——修改文件数、新增行数（+）、删除行数（−）——以及一条建议的提交信息，配「复制」按钮，一键拿去 commit。

9 · 权限审批与命令风险分级

审批是本产品最核心的交互。Agent 请求执行命令时，卡片展开审批区，显示命令原文（等宽、可换行）、Allow / Deny 按钮，并按命令内容做风险分级：

等级 判定规则（正则匹配命令） 处理
⚠ 危险rm -rf、git push --force、sudo、dd if=、mkfs、chmod 777、> /dev/、del /f、format、DROP/TRUNCATE TABLE、shutdown/reboot 等命令标红；Allow 键需两步确认（先点武装、再点「确认执行」才放行）。
✓ 安全ls、cat、pwd、echo、npm/yarn/pnpm test、pytest、jest、git status/diff/log、grep、rg、find 等可勾选「本会话自动允许只读 / 测试类命令」，后续同类请求自动放行并提示。
· 一般其余命令常规 Allow / Deny。

审批区还展示等待计时（见 §14）与快捷键提示（Y 允许 / N 拒绝）。拒绝后自动展开接力对话，便于说明理由。

10 · 对话接力与富输入

接力对话让用户无需切回终端即可与 Agent 继续交互。每张 Agent 卡片、以及 Plan 分区，都内嵌一个统一的对话组件：消息按用户/Agent 分气泡展示，Agent 回复以结构化排版呈现（小标题、要点列表、代码块、提示），便于工程师快速阅读。

富输入（三种）：

  • 文字——回车发送；
  • 图片（🖼）——选择图片，生成缩略图附件；
  • 文件（📎）——选择任意文件作为附件，图片类自动出缩略图。

设计沿革：早期版本含语音输入，但其形态被调整为「说完自动转文字填入输入框」；后因交互存在闪烁问题、且非核心诉求，本期已移除语音与主动截图，仅保留文字 / 图片 / 文件三种输入。

11 · Plan 审阅

Plan 分区渲染 Agent 提交的实施计划：标题、Markdown 描述、分步清单（已完成 ✓ / 进行中 ◐ / 待办 ○ 三态图标），底部提供「批准计划 / 批准并执行」，以及内嵌的接力对话——用户可直接提修改意见（配快捷回复气泡：批准 / 再细化第 2 步 / 拆得更小些），Agent 给出修订后的结构化方案。

12 · 问答助手与模型设置

除监控主目标外，岛还提供一个知识问答助手：工作中遇到问题随手提问，回答以结构化排版给出，支持文字/图片/文件输入。空态提供引导与一键建议问题；分区顶部实时显示当前模型，点击直达设置。

模型设置（仅供问答助手，折叠展开式）：Agent（Claude Code / Codex）的模型在各自 CLI 里已配置，此处不涉及。设置项包含：

  • 供应商——DeepSeek / Kimi / 通义千问 / GPT(OpenAI) / Claude / 自定义；切换只带出对应 Base URL。
  • 型号——用户自行填写 model id（不做推荐）。
  • Base URL、API Key——兼容 OpenAI Chat Completions 协议的服务均可接入。
  • 测试连通性——校验字段并给出成功/失败反馈（带加载动画）。
  • 多套配置——「保存当前」存为一套，列表内一键切换/删除，当前使用中高亮标注。

密钥仅保存在本机。设计原型中问答走内置模型演示；正式 Windows 客户端将直连用户配置的 Base URL / Key。

13 · 快捷键与快捷回复

按键 行为
Y允许第一个待处理请求
N拒绝第一个待处理请求（并展开接力对话）
Esc收起面板（未贴住时）
Ctrl + \ / `唤出 / 收起面板（全局热键）

输入框上方提供快捷回复气泡，一键发送常用回复（接力：继续 / 换个方案 / 先跑测试再说；Plan：批准 / 再细化 / 拆更小些）。焦点在输入框时，Y/N 等快捷键自动让位于打字。

14 · 通知、声音与久等提醒

  • 完成通知——即使在静默态，Agent 完成任务也会从顶部滑出一个通知气泡（可关闭、自动消失）。
  • 终端跳转反馈——点击跳转后底部出现「已聚焦到 XXX 的终端窗口」浮条。
  • 声音提醒——需要处理时播放提示音；音效在「声音提醒」开关下方内联展开选择（清脆铃声 / 8-bit / 单声 Ping / 木琴 / 警示音），每项可试听，选完自动收起。音效用 WebAudio 实时合成，无需音频文件。
  • 久等提醒——权限请求挂起超过阈值（演示为 20s）后，胶囊对应状态点从琥珀升级为红色并加快呼吸，审批区显示「⏱ 已等待 M:SS」，避免遗忘。

15 · 专注模式与贴住

专注模式（🌙）——一键开启 25 分钟静默倒计时。期间新的权限请求不弹窗、不高亮、不发声，只在胶囊上安静堆叠计数；倒计时结束时一次性弹出汇总（「专注结束 · 期间累积 N 个待处理请求」）。适合番茄钟式深度工作。

贴住（📌）——见 §5，让胶囊常驻不收起。

16 · 拖拽投喂

把桌面上的图片或文件直接拖到面板上，面板出现「📥 松手投喂到问答助手」的高亮虚线覆盖层；松手后文件作为附件进入「问答」输入框并自动切到该分区。降低「带着一个文件去提问」的操作成本。

18 · 技术架构（Windows 实现建议）

本设计以高保真交互原型交付。落地为 Windows 原生客户端时的建议架构：

模块 实现要点
窗口与渲染WinUI 3 / WPF 无边框置顶层窗口，DWM 亚克力/Mica 实现毛玻璃；始终置顶、点击穿透空白区、跨虚拟桌面可见。
顶部感应区全局低级鼠标钩子（SetWindowsHookEx / WH_MOUSE_LL）监听光标进入顶部中央热区；多显示器按光标所在屏定位。
全局热键RegisterHotKey 注册唤出热键；面板内 Y/N/Esc 走窗口消息。
Agent 接入通过各 CLI 的 hook / 事件流（stdout 解析或官方钩子）捕获权限请求、计划、完成事件；桌面端走其本地 API / IPC。
终端跳转记录会话与终端窗口/进程的映射（Windows Terminal、ConEmu、WSL、tmux 等），跳转时 SetForegroundWindow 并定位到对应 pane。
问答模型代理本地转发到用户配置的 OpenAI 兼容端点；密钥用 Windows Credential Manager / DPAPI 加密存储。
声音WebAudio 或原生音频合成 5 种提示音；无需打包音频资源。

原型实现要点（值得研发注意的一个坑）：面板展开的 max-height 过渡动画会被「每秒刷新一次的等待计时器」重渲染打断而卡住。解决方案是让面板展开为即时（不对 max-height 做过渡），并把逐秒计时器限制为仅在有待处理请求或专注模式时才触发重渲染。原生实现中同理：驱动动画的时钟应与状态计时解耦。

19 · 状态机与交互时序

岛的可见性由三个输入决定：悬停/焦点、是否有待处理、是否贴住；并受专注模式抑制：

  • 显示 = 悬停 || (有待处理 && 非专注) || 贴住
  • 展开 = 已点击展开 || (有待处理 && 非专注 && 已显示)
  • 收起：点击外部 / 再点胶囊 / Esc；但「有待处理」「贴住」「焦点在输入框」时不自动收起。

危险命令审批为两步状态：待确认 → 已武装 → 放行；决策后清除该请求的等待计时与武装态。

20 · 数据、隐私与安全

  • 本地优先——模型密钥、配置、会话历史均仅存于本机，不上云。
  • 密钥加密——API Key 以 DPAPI / Credential Manager 加密存储，UI 中以密码形式呈现。
  • 危险操作护栏——破坏性命令强制二次确认，降低误触发风险。
  • 最小权限——仅请求实现监控/跳转所需的系统权限（窗口枚举、前台切换、全局钩子）。
